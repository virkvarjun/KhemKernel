## Why from scratch

PyTorch and JAX make transformers easy in a way that hides whether you
understand them. Re-implementing in NumPy is a forcing function: every
broadcast, every gradient, every numerical quirk has to be confronted
explicitly.

## The ops library

A small set of differentiable primitives:

```python
def linear(x, W, b):
    return x @ W + b

def linear_backward(dY, x, W):
    dW = x.T @ dY
    db = dY.sum(axis=0)
    dX = dY @ W.T
    return dX, dW, db
```

Each op has a paired backward function. The training loop walks the graph
manually — slow, but unambiguous.

## Attention

Scaled dot-product attention is three matmuls and a softmax. The gotcha is
the backward pass through softmax: it's a Jacobian-vector product, not just
elementwise.

<Footnote id="1">The Jacobian of softmax is dense: <code>∂s_i/∂x_j = s_i(δ_ij − s_j)</code>. The backward op collapses this to a single contraction.</Footnote>

## Overfit test

The classic sanity check: take a single batch, train it for 1,000 steps,
and confirm the loss collapses near zero. If it doesn't, the gradients are
wrong.
