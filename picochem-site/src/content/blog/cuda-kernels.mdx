## GPU basics

A short refresher on the bits that actually matter for kernel writing:
warps, shared memory, the difference between global and local memory, and
the L2 cache.

## Vector add

The "hello world" kernel:

```cpp
__global__ void vector_add(const float* a, const float* b, float* c, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) c[i] = a[i] + b[i];
}
```

This already gets you within shouting distance of memory bandwidth on small
arrays. The interesting thing is what changes when you go bigger.

## Tiled matmul

Naive matmul is bandwidth-limited. A tiled version that stages submatrices
through shared memory turns it into an arithmetic-intensity problem
instead.<Footnote id="1">For a tile of size T×T, each global load is reused T times. The compute-to-memory ratio scales with T, until you run out of registers.</Footnote>

Plan to walk through: tile size selection, padding for bank conflicts,
double-buffering, and the surprising amount of performance left on the
table by *not* using tensor cores.
